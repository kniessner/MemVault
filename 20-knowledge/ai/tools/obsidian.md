# Obsidian

Obsidian is the local knowledge environment for this vault. It should be used as a durable note graph, not as a random file pile.

## Why it matters for AI work

- properties make AI notes queryable and structured
- wikilinks and embeds make relationships explicit
- templates reduce note-shape drift
- CLI and URI flows make automation practical
- plugin ecosystem can extend workflows, but core features should remain the default foundation

## Preferred vault usage

### Capture

Use `10-notes/40-resources/` for raw external research capture.

### Distill

Move stable insights into `20-knowledge/ai/` by provider, model, tool, or concept.

### Operate

Use links, properties, and templates so agent outputs remain easy to find, query, and review.

## Practical rule

Do not leave durable AI knowledge trapped inside session logs, chat exports, or one-off project files if the knowledge is reusable.
