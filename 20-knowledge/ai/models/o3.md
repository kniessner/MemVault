# o3

> OpenAI's most powerful reasoning model — excels at complex problem-solving, coding competitions, and scientific reasoning.

## Overview

o3 is OpenAI's latest reasoning model, designed specifically for tasks requiring deep thinking, complex logic, and multi-step problem-solving. It uses "deliberative alignment" to reason through safety guidelines before responding.

## API Example

```python
from openai import OpenAI

client = OpenAI()

# o3-high reasoning effort
response = client.chat.completions.create(
    model="o3",
    reasoning_effort="high",  # low, medium, high
    messages=[{
        "role": "user",
        "content": """Solve this step by step:
        A bakery sells muffins in packs of 3 and 5. 
        What's the largest number of muffins you CANNOT buy exactly?"""
    }]
)

print(response.choices[0].message.content)
print(f"Reasoning tokens: {response.usage.completion_tokens_details.reasoning_tokens}")
```

### Reasoning Effort Settings

| Setting | Use Case | Latency | Cost |
|---------|----------|---------|------|
| **low** | Quick reasoning | Fast | Lower |
| **medium** | Balanced | Medium | Medium |
| **high** | Maximum quality | Slower | Higher |

- ❌ Cost-sensitive workloads

## Deliberative Alignment

Unique safety approach: reasons through safety principles before responding.

```
User: "How do I hack a website?"

o3 thinking:
1. User is asking about unauthorized access
2. This violates computer fraud laws
3. I should decline and explain why
4. I can offer legitimate security education instead

Response: "I can't help with unauthorized access..."
```

## Limitations

1. **Price** — 10x more expensive than GPT-4o
2. **Speed** — Can take 10-60 seconds for complex problems
3. **No vision** — Text only
4. **Limited tool use** — Different from GPT-4o function calling
5. **Rate limits** — Strict quotas, limited availability
6. **Output length** — Max 100K tokens (vs 4K for o1)
